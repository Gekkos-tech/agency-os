# Phase 2 — CONCEPT

Goal: decide what the site says and how it is organized, before anything is
designed. Output: sitemap + page narratives + creative direction.

## Work

1. **Sitemap** — pages and their hierarchy, derived from the brief's user
   tasks. Fewer pages beat more: every page must earn its place by serving a
   task or the goal.
2. **Page narrative per page** — the argument the page makes, section by
   section (e.g. hero → proof → offer → objection handling → CTA). One line
   per section; no copy yet, but name each section's job.
3. **Creative direction** — load `taste` if installed: infer the direction
   from the brief (who is this for, what should it feel like, what would the
   generic version look like — then avoid that). If the client has a brand,
   check `awesome-design-md` for comparable reference systems. Capture the
   direction in 5 lines: mood, palette family, type feel, motion character,
   one distinctive signature idea.
4. **Conversion path** — from entry to primary goal in the fewest steps;
   name the CTA per page. For campaign-driven projects, load `ad-concepts`
   to align site messaging with the campaign angle.

## Output document

`docs/concept.md`: sitemap (tree), page narratives, creative direction,
conversion path, and any open content gaps that block DESIGN.

## Exit gate

- [ ] Every brief user-task is served by a page or section
- [ ] Each page has a stated job and a primary CTA
- [ ] Creative direction is specific enough that two designers would produce
      similar moods from it
- [ ] Client approved sitemap + direction — or user waived approval
