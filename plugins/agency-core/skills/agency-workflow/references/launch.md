# Phase 6 — LAUNCH

Goal: go live safely and hand the project over cleanly.

## Work

1. **Launch gate** — run `pre-launch-checklist` if installed (legal +
   technical launch requirements; for DACH clients: DSGVO, Impressum,
   Cookie-Consent, BFSG accessibility). No launch with open legal items.
2. **SEO baseline** — run the installed SEO audit skill once against the
   staging URL: indexability, sitemap, robots, canonical, meta/OG tags,
   schema for the business type (LocalBusiness for local clients).
3. **Go-live** — DNS/hosting cutover plan with rollback; SSL verified;
   redirects from any old URLs mapped 1:1 (relaunch projects: crawl the old
   site first, map every indexed URL); custom 404 in place.
4. **Post-launch verification** — within the first hour: live URL loads on
   mobile + desktop, forms deliver, analytics/consent fire correctly,
   Search Console sees the sitemap.
5. **Handover** — deliver to the client: access credentials location, CMS
   how-to (if any), the QA report, and maintenance recommendations. For
   campaign launches, produce launch assets via `social-formats` and
   `ad-copy`.

## Exit gate

- [ ] Pre-launch checklist passed (legal items = hard blockers)
- [ ] Redirect map live and spot-checked (relaunches)
- [ ] Post-launch verification done on the live domain
- [ ] Handover document delivered
- [ ] Project retro: one paragraph — what to reuse, what to avoid next time
